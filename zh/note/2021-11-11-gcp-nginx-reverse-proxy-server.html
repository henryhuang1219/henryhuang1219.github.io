GCP - Nginx 反向代理

Henry Huang 11/11/2021 Google Cloud PlatformNginxMySQL

原本是將 Server 架在實機上，由於在外租屋固定 IP 不好處理及電費因素考量下，決定改為使用 GCP 的 Compute Engine。 將架設過程中所使用到的指令及套件稍作整理記錄下來。

# Google Cloud Platform

# 建立 Compute Engine

這裡依照 Google Cloud Engine 免費方案的地區以及機器設定配置。

開機磁碟配置。

10 GB 在安裝完 OneinStack，所剩的儲存空間會不足後續使用。

# 設定 SSH

創建並取得 SSH 公鑰：

mkdir ~/.ssh
ssh-keygen -t rsa -f ~/.ssh/KEY_FILENAME -C USER_NAME -b 2048
cat ~/.ssh/KEY_FILENAME.pub

ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQDAu5kKQCPF... cloudysanfrancisco

進入 VM 詳細資訊，點擊編輯，加入 SSH 公鑰。

測試 SSH 連線：

ssh -i ~/.ssh/KEY_FILENAME USER_NAME@EXTERNAL_IP

# Ubuntu

# 設定 root

避免執行某些命令時需要一直重複加上sudo的這個困擾，在開始前即可先初始化 root 的密碼：

sudo passwd root

切換至 root：

su -

# OneinStack

至 OneinStack 官網選擇自動安裝 (opens new window)，依照自己需求選擇預設配置，可再針對預設配置做進一步的編輯。

執行 OneinStack 所產生的命令，需等待約 1 個小時左右：

wget -c http://mirrors.linuxeye.com/oneinstack-full.tar.gz && tar xzf oneinstack-full.tar.gz && ./oneinstack/install.sh --nginx_option 1 --db_option 2 --dbinstallmethod 1 --dbrootpwd YOUR_ROOT_PASSWORD --reboot

需在 root 權限下執行。

# JDK

# 上傳 JDK

下載 JDK 1.8 (opens new window)，選擇 Linux x64 Compressed Archive。

使用 SCP 將檔案上傳至 GCP：

scp -i ~/.ssh/KEY_FILENAME ~/Downloads/jdk-8u301-linux-x64.tar.gz USER_NAME@EXTERNAL_IP:~

# JDK 配置

解壓縮 tar.gz：

tar -zxvf jdk-8u301-linux-x64.tar.gz

移動 JDK 資料夾至 /usr/local/jdk/ 之下：

mkdir /usr/local/jdk
mv jdk1.8.0_301 /usr/local/jdk/

新增捷徑目錄：

cd /usr/local/
ln -s jdk/jdk1.8.0_301/ java

之後 JDK 更版只要把 java 捷徑指向不同路徑即可，不再需要去調整環境變數。

加入環境變數設定：

vim /etc/profile

...

export JAVA_HOME=/usr/local/java
export JRE_HOME=$JAVA_HOME/jre
export PATH=$JAVA_HOME/bin:$JRE_HOME/bin:$PATH
export CLASSPATH=$CLASSPATH:.:$JAVA_HOME/lib:$JRE_HOME/lib

查看 Java 版本：

java -version

Java(TM) SE Runtime Environment (build 1.8.0_202-b08)
Java HotSpot(TM) 64-Bit Server VM (build 25.202-b08, mixed mode)

# 創建 Virtual Host

確認 Nginx 正常運行後，便可在 http://EXTERNAL_IP/###vhost (opens new window) 網頁中查看教學。

新增完 Virtual Host 的域名後，記得去設定對應的 CNAME。

# 設定 Reverse Proxy

修改 Virtual Host 的設定檔：

vim /usr/local/nginx/conf/vhost/DOMAIN_NAME.conf

server {
  listen 80;
  listen [::]:80;
  server_name DOMAIN_NAME;
  access_log off;
  
  location ~ /(\.user\.ini|\.ht|\.git|\.svn|\.project|LICENSE|README\.md) {
    deny all;
  }
  location ~ {
    proxy_pass http://127.0.0.1:SERVER_PORT_NUMBER;
  }
}

SERVER_PORT_NUMBER 即是所架設服務的 Port 號。 例如：Django 預設為 8000，根據專案配置設定 Port 號。

更改完成後，重新啟動 Nginx：

service nginx restart
